Runtime modes
=============

1. Mock mode - no server and no database required
   - Run main class: com.auction.client.ClientApp
   - Add VM option:
     -Dapp.mock=true
   - Client uses MockDataProvider for login/dashboard/demo data.
   - Registration is treated as a demo success and is not persisted.

2. Real server + database mode
   - Do not use -Dapp.mock=true.
   - Start MySQL and import SQL from database/schema.sql and database/init_data.sql.
   - Confirm server/src/main/resources/database.properties:
     db.url=jdbc:mysql://localhost:3306/auction
     db.username=root
     db.password=
     db.driver=com.mysql.cj.jdbc.Driver
   - Start server main class:
     com.auction.server.ServerApp
   - Start client main class:
     com.auction.client.ClientApp

Important notes
===============

- ClientApp is the class to run from IntelliJ. AuctionClientApplication is the JavaFX Application class used internally.
- The JavaFX resources are currently stored under client/src/main/java/com/auction/client/view and styles under client/src/main/java/com/auction/client/styles.
- The project currently mixes text TCP commands and JSON message helpers. ServerConnection supports the text command flow used by ClientHandler.
- Some DAO classes are still in-memory oriented. DatabaseConnection and database.properties exist for JDBC configuration, but persistence coverage depends on each DAO implementation.
